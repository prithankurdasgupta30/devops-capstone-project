# Contributing to Simple Interest Calculator

Thank you for taking the time to contribute! We welcome all contributions that help improve this project, from fixing typos to adding new calculator features.

By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md).

## How Can I Contribute?

### Reporting Bugs
If you find a bug or unexpected behavior in the script:
1. Check the **Issues** tab to see if it has already been reported.
2. If not, open a new issue.
3. Clearly describe the problem, include steps to reproduce it, and paste any error messages you received.

### Suggesting Features
Have an idea to make this calculator better? 
1. Open a new issue.
2. Explain the feature you want to add (e.g., adding compound interest calculation, building a web UI).
3. Explain why this feature would be useful to other users.

### Submitting Code Changes
Ready to jump in and write code? Please follow these steps:

1. **Fork the Repository**: Create your own copy of the project by clicking the "Fork" button at the top of the GitHub page.
2. **Clone Locally**: Clone your fork to your computer.
   ```bash
   git clone https://github.com
   ```
3. **Create a Branch**: Create a new branch for your feature or fix. Use a descriptive name.
   ```bash
   git checkout -b feature/add-compound-interest
   ```
4. **Make Your Changes**: Write your code and ensure it follows clean Python formatting. 
5. **Test Your Changes**: Run the script locally to make sure your changes work and do not break existing features.
6. **Commit and Push**: Commit your changes with a clear message and push them to your fork.
   ```bash
   git add .
   git commit -m "Add compound interest calculation option"
   git push origin feature/add-compound-interest
   ```
7. **Open a Pull Request**: Go back to the original repository on GitHub. You should see a prompt to open a "Pull Request". Submit it with a detailed explanation of your changes.

## Code Style Guide
To keep the project clean and readable, please make sure your Python code:
- Follows standard **PEP 8** style guidelines.
- Uses clear, descriptive variable names (e.g., `interest_rate` instead of `ir`).
- Includes basic comments or docstrings for new functions.

## Questions?
If you have any questions or get stuck, feel free to open an issue or reach out via the email listed in our Code of Conduct.
